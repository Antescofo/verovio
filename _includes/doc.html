This page documents the elements and attributes that are currently supported by Vevorio. It is also important to note that the hierarchy of elements supported by Verovio is limited and that it will also determine wherether an MEI document can be rendered or not.

Page-based customization elements

A few elements supported by Verovio are defined in the page-based customization describe on the structure page.

<page>
Attribute Comment Att-class
@page.height
@page.leftmar
@page.rightmar
@page.topmar
@page.width
@surface

Several attributes of <page> are stored in the <scoreDef> element in uncustomized MEI.

<secb>

Currently, <secb> elements are not handled.

<system>
Attribute Comment Att-class
@system.leftmar
@system.rightmar
@uly This is used for positionned page-based encoding.

Several attributes of <system> are stored in the <scoreDef> element in uncustomized MEI.

Load-only MEI elements

Verovio can load both page-based customized and uncustomized MEI documents. When loading uncustomized MEI documents, some MEI elements are loaded by Verovio and converted to a page-based representation. Typically, <pb> milestone elements are converted to <page> container elements. Oppositely, <section> container elements are converted to <secb> milestone elements.

There are also some MEI elements that Verovio can load but only if they can be converted to an alternate MEI representation. For example, Verovio can load <tupletSpan> elements but only when then can be appropriately converted to <tuplet> elements. Verovio tries to perfom the convertions when loading the document. This has nothing to do with the page-based customization.

<pb>

During import, <pb> elements are converted to <page> elements unless the --ignore-layout or the --no-layout option was selected.

<sb>

During import, <sb> elements are converted to <system> elements unless the --ignore-layout or the --no-layout option was selected.

<score>
<section>

During import, <section> elements are converted to <secb> elements.

<tupletSpan>

During import and whenever possible, <tupletSpan> elements are converted to <tuplet> elements.
